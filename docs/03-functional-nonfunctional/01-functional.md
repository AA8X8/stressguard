## UC-1: Регистрация и первичная настройка (Onboarding)

| Раздел | Описание |
|--------|----------|
| **ID** | UC-1 |
| **Название** | Регистрация и подключение источников данных |
| **Акторы** | Пользователь (сотрудник/студент), Google Fit API / Apple HealthKit, Google Calendar API / Microsoft Graph |
| **Предусловия** | Пользователь установил мобильное приложение, устройство имеет доступ к интернету |
| **Постусловия (Успех)** | Учётная запись пользователя создана, подключены календарь и носимые устройства (по выбору), пользователь проинформирован о системе плюшек и необходимости заполнять календарь |
| **Постусловия (Неудача)** | Учётная запись не создана или источники не подключены, отображается сообщение об ошибке с предложением повторить |

**Основной поток:**
1. Пользователь открывает приложение и нажимает «Зарегистрироваться».
2. Система предлагает ввести email и пароль или войти через Google/Apple.
3. Пользователь вводит данные и подтвержда регистрацию.
4. Система запрашивает разрешение на доступ к календарю (Google/Apple/Outlook).
5. Пользователь предоставляет разрешение. Система проверяет, что календарь не пуст (есть хотя бы одно событие в ближайшие 7 дней). Если пуст, выводит предупреждение: «Для точной оценки стресса добавьте события в календарь».
6. Система запрашивает разрешение на доступ к данным о здоровье (Google Fit / Apple Health).
7. Пользователь предоставляет разрешение.
8. Система отображает приветственный экран с краткой информацией о плюшках:
   - Студенты: 50 входов за 3 месяца → автомат по одному предмету.
   - Сотрудники: 80 входов за 5 месяцев → +1 день к отпуску.
9. Система сообщает, что каждый вход и ввод данных о самочувствии приближают к цели.
10. Регистрация завершена, пользователь переходит на главный экран.

**Альтернативный поток 1 (отказ в доступе):**
- Точка входа: шаг 4 или 6
- Пользователь отказывает в доступе к календарю.
- Система уведомляет: «Без календаря мы не сможем учитывать вашу нагрузку. Вы можете подключить его позже в настройках».
- Продолжение с шага 6 или завершение регистрации.

**Бизнес-правила:**
- BR-01: Календарь пользователя должен содержать хотя бы одно событие в ближайшие 7 дней для корректной работы рекомендаций (проверяется при подключении и далее еженедельно).
- BR-02: Доступ к данным здоровья необходим для анализа физиологических показателей. При отсутствии доступа система работает в урезанном режиме.

## UC-2: Ежедневный ввод данных о самочувствии

| Раздел | Описание |
|--------|----------|
| **ID** | UC-2 |
| **Название** | Ежедневный ввод данных о самочувствии |
| **Акторы** | Пользователь (сотрудник/студент) |
| **Предусловия** | Пользователь зарегистрирован и выполнил вход, приложение открыто (или пользователь перешёл по уведомлению) |
| **Постусловия (Успех)** | Введены данные о настроении, усталости, качестве сна, пользователю начислены баллы (плюшки), данные сохранены для анализа |
| **Постусловия (Неудача)** | Данные не сохранены, отображается сообщение об ошибке |

**Основной поток:**
1. Пользователь открывает приложение (или нажимает на уведомление «Как вы сегодня?»).
2. Система отображает экран «Ваше самочувствие».
3. Пользователь выбирает уровень настроения (шкала от 1 до 5, где 1 – очень плохо, 5 – отлично).
4. Если выбран уровень ≤ 3, система предлагает указать причину (один из вариантов: конфликты/ссоры, погода, здоровье, сложности в работе/учёбе, другое).
5. Пользователь выбирает причину (или вводит свою).
6. Система переходит к экрану «Усталость»: пользователь оценивает уровень усталости (ползунок от 1 до 5).
7. Система переходит к экрану «Сон»: пользователь отмечает параметры сна:
   - Спокойный / Неспокойный
   - Менее 8 часов / 8 часов / более 8 часов
   - Засыпание: быстро / долго
8. Пользователь подтверждает ввод.
9. Система сохраняет все ответы, связывает их с временной меткой и начисляет пользователю +1 балл за полный цикл ввода.
10. Система отображает сообщение: «Спасибо! +1 балл. Вы приближаетесь к цели: X/Y входов».

**Альтернативный поток (прерывание ввода):**
- Пользователь закрывает приложение до завершения.
- Система сохраняет только введённые на данный момент данные (если возможно) и не начисляет баллы.
- При следующем входе система может напомнить о незавершённом вводе.

**Бизнес-правила:**
- BR-03: За каждый полный цикл ввода (настроение + усталость + сон) начисляется 1 балл. Частичный ввод баллов не приносит.
- BR-04: Причина плохого настроения фиксируется только если оценка ≤ 3.

## UC-3: Просмотр главного экрана (дашборда)

| Раздел | Описание |
|--------|----------|
| **ID** | UC-3 |
| **Название** | Просмотр главного экрана (дашборда) |
| **Акторы** | Пользователь (сотрудник/студент) |
| **Предусловия** | Пользователь зарегистрирован и выполнил вход, приложение открыто (главный экран) |
| **Постусловия (Успех)** | Пользователь видит актуальную информацию: текущий уровень стресса, график за неделю, список персонализированных рекомендаций |

**Основной поток:**
1. Пользователь открывает приложение (или переходит на главный экран из другого раздела).
2. Система загружает последние данные о состоянии пользователя:
   - стресс-индекс (рассчитанный ML-модулем)
   - историю стресс-индекса за последние 7 дней
   - список актуальных рекомендаций
3. Система отображает главный экран, содержащий:
   - Индикатор текущего уровня стресса (цветовой круг или шкала: зелёный – низкий, жёлтый – средний, красный – высокий)
   - Краткую историю изменений (график динамики стресс-индекса за последние 7 дней)
   - Блок «Рекомендации» (список из 1–3 персонализированных советов)
4. При нажатии на рекомендацию пользователь может увидеть её полное описание и/или перейти к действию.
5. При нажатии на график или индикатор пользователь может перейти к экрану «Подробная история».
6. Система автоматически обновляет данные на главном экране при каждом открытии или по запросу пользователя.

**Альтернативный поток (отсутствие данных):**
- Если за последние 7 дней недостаточно данных для построения графика, система отображает заглушку: «Накопление данных… Появятся после нескольких дней использования».
- Если нет актуальных рекомендаций, система отображает ободряющее сообщение: «У вас всё хорошо! Продолжайте в том же духе».

## UC-4: Отслеживание прогресса получения плюшек

| Раздел | Описание |
|--------|----------|
| **ID** | UC-4 |
| **Название** | Просмотр прогресса накопления плюшек |
| **Акторы** | Пользователь |
| **Предусловия** | Пользователь зарегистрирован и выполнил вход |

**Основной поток:**
1. Пользователь открывает раздел «Мои плюшки».
2. Система отображает:
   - Текущее количество входов (например, 5 из 50 для студентов, 15 из 80 для сотрудников)
   - Оставшееся время для достижения цели
   - Прогресс-бар
   - Описание награды
3. Если цель достигнута, система отображает поздравительное сообщение и инструкцию.
4. При нажатии кнопки «Подробнее» показывается история входов по дням.

**Бизнес-правила:**
- BR-07: Засчитываются только те входы, в которых пользователь провёл в приложении не менее 30 секунд и/или выполнил ввод данных.
- BR-08: Цели различаются для студентов и сотрудников.

## UC-5: Просмотр агрегированной статистики по отделам/группам для HR/деканата

| Раздел | Описание |
|--------|----------|
| **ID** | UC-5 |
| **Название** | Просмотр агрегированной статистики по отделам/группам |
| **Акторы** | HR-специалист (для организаций), сотрудник деканата (для вузов) |
| **Предусловия** | Представитель HR/деканата авторизован в веб-дашборде, в системе зарегистрированы пользователи, относящиеся к данной организации |

**Основной поток:**
1. HR-специалист входит в веб-дашборд и выбирает раздел «Аналитика».
2. Система отображает главную страницу аналитики с предустановленными фильтрами: «Все отделы/группы» и период «Последние 30 дней».
3. На странице представлены блоки:
   - Средний уровень стресса по организации/вузу
   - Динамика среднего стресса за период (линейный график)
   - Тепловая карта стресса по отделам/группам
   - Активность пользователей (доля активных, количество зарегистрированных)
   - Список достижений (плюшки) с количеством достигших цели
4. HR-специалист может применить фильтры (отдел/группа, период, тип данных).
5. После применения фильтров система обновляет все блоки.
6. HR-специалист может нажать кнопку «Экспорт отчёта» и выбрать формат (PDF или Excel).

**Бизнес-правила:**
- BR-14: Все данные отображаются только в агрегированном и обезличенном виде.
- BR-15: Доступ к статистике имеют только авторизованные представители организации/вуза.
- BR-16: Экспортируемые отчёты также не содержат персональных данных (за исключением списка достигших целей).

## UC-6: Просмотр статистики достижений пользователей для HR/деканата

| Раздел | Описание |
|--------|----------|
| **ID** | UC-6 |
| **Название** | Просмотр статистики достижений пользователей |
| **Акторы** | HR-специалист, сотрудник деканата |
| **Предусловия** | Представитель HR/деканата авторизован в веб-дашборде, есть пользователи, достигшие целей по плюшкам |

**Основной поток:**
1. HR-специалист входит в веб-дашборд и выбирает раздел «Награды».
2. Система отображает две вкладки: «Студенты» и «Сотрудники».
3. На каждой вкладке показан список пользователей, достигших цели за текущий период.
4. Для каждого пользователя указаны:
   - ФИО (или идентификатор)
   - Количество входов
   - Дата последнего входа
   - Статус награды (ожидает выдачи / выдана)
5. Рядом с каждым пользователем есть кнопка «Отметить как выданное».
6. При нажатии система запрашивает подтверждение и меняет статус.
7. Также доступен экспорт списка в Excel/PDF для оформления приказов.

**Бизнес-правила:**
- BR-09: Доступ к статистике имеют только авторизованные представители организации.
- BR-10: Информация о пользователях отображается обезличенно (только ФИО и прогресс, без привязки к личным данным о здоровье).
