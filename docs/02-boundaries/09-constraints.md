## Бюджет

### В рамках учебного проекта

- Прямое финансирование отсутствует. Все работы выполняются студентом (или командой студентов) в рамках дипломного проектирования.
- Используются бесплатные тарифы облачных сервисов, инструментов разработки и API (Firebase, Google Fit API, Apple HealthKit, PostgreSQL, Python-библиотеки с открытым исходным кодом и т.п.).
- Затраты ограничены личным временем разработчика и ресурсами учебного заведения (например, доступ к лабораториям, консультации преподавателей).

### При гипотетическом коммерческом запуске

- На этапе MVP (6 месяцев) потребуется бюджет в размере **500 000 – 700 000 рублей**, включающий:
  - Оплату облачной инфраструктуры (серверы, базы данных, CDN) – 50 000 руб./мес. (первые 6 месяцев = 300 000 руб.).
  - Покупку домена, SSL-сертификатов, аккаунтов разработчика в App Store и Google Play (единоразово около 15 000 руб.).
  - Маркетинговые расходы на привлечение пилотных клиентов (контент, презентации, участие в профильных мероприятиях) – 100 000 руб.
  - Резерв на непредвиденные расходы – 85 000 руб.
- Для дальнейшего развития (версии 1.1, 2.0) бюджет должен увеличиваться за счёт привлечения инвестиций или первых поступлений от клиентов.

## Сроки

Общий срок реализации MVP: **6 месяцев** с даты старта проекта.

### Ключевые вехи (этапы) проекта

| Веха | Срок от старта | Результат |
|------|----------------|-----------|
| Анализ и концепция | 1 месяц | Утверждённая концепция, список стейкхолдеров, бизнес-требования, user story map. |
| Проектирование и дизайн | 2 недели | Прототипы интерфейсов (мокапы), архитектурная схема, ER-диаграмма, спецификация API. |
| Разработка бэкенда | 2,5 месяца | Готовый бэкенд с API, интеграциями (Google Fit, календари), ML-модулем, базой данных. |
| Разработка фронтенда (мобильное приложение) | 2,5 месяца (параллельно с бэкендом) | Рабочее мобильное приложение для iOS/Android с основным функционалом. |
| Разработка HR-дашборда (веб) | 1 месяц (после бэкенда) | Веб-интерфейс для HR-специалистов с базовой аналитикой. |
| Тестирование и отладка | 2 недели | Исправление критических ошибок, подготовка к пилоту. |
| Запуск MVP и пилотный проект | 6 месяцев | Релиз приложения, подключение 2–3 пилотных организаций/вузов. |
| Сбор обратной связи и метрик | 7–8 месяцев | Анализ результатов пилота, корректировка дорожной карты. |

> **Примечание:** этапы могут частично пересекаться при использовании гибких методологий (Agile).

## Ресурсы

### Человеческие ресурсы (команда)

В коммерческом проекте минимальный состав команды на этапе MVP — **3 человека**: аналитик/PM, backend-разработчик, frontend-разработчик. Тестирование и дизайн могут быть на аутсорсе или выполняться разработчиками.

### Технологические ресурсы (стек)

| Компонент | Технологии / Инструменты | Обоснование |
|-----------|--------------------------|-------------|
| Бэкенд | Python (FastAPI / Django), PostgreSQL, TimescaleDB (для временных рядов), Redis (кэш, очереди) | FastAPI — высокая производительность, асинхронность, удобная документация. PostgreSQL — надёжность, поддержка JSON. TimescaleDB — оптимизация для метрик. |
| ML-модель | Python (scikit-learn, pandas, numpy), ONNX (для развёртывания) | Широкий выбор библиотек, возможность быстрого прототипирования. |
| Мобильное приложение | React Native (iOS 14+, Android 10+), JavaScript/TypeScript | Кроссплатформенная разработка, единая кодовая база, знание стека командой. |
| HR-дашборд (веб) | React / Vue.js | Современные фреймворки для интерактивных интерфейсов. |
| Интеграции | Google Fit API, Apple HealthKit, Google Calendar API, Microsoft Graph API | Официальные API провайдеров. |
| Инфраструктура | Docker, Kubernetes (опционально), облачные платформы (AWS / GCP / Yandex.Cloud) | Контейнеризация упрощает развёртывание, облако — масштабирование. Для учебного проекта можно использовать виртуальную машину или Heroku-подобные сервисы. |
| Уведомления | Firebase Cloud Messaging (Android), APNs (iOS) | Стандартные платформенные сервисы. |
| Система контроля версий | Git (GitHub / GitLab) | Стандарт индустрии. |
| Управление проектом | Jira / Trello / YouTrack | Отслеживание задач, бэклог. |

### Прочие ресурсы

- Доступ к тестовым аккаунтам Google/Apple для отладки интеграций.
- Мобильные устройства для тестирования (Android и iOS).
- Время преподавателя/консультанта для ревью.

> Данные ограничения являются основой для планирования и управления ожиданиями стейкхолдеров. В процессе реализации возможна корректировка при условии согласования с заказчиком.
